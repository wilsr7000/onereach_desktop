## Critical / Blocking

### Pre-Public-Launch (must fix before non-pilot users)
- [ ] **Remove the baked-in default OAGI / Neon graph from source.** `lite/neon/credentials.ts` exports `BAKED_IN_DEFAULT_GRAPH` containing the team's shared Aura URI / user / password / database plus the well-known Edison `/omnidata/neon` endpoint. The production singleton in `lite/neon/api.ts` wires this as the `KVCredentialsProvider.fallbackRecord` so fresh installs report `ready: true` and queries connect without the user touching Settings -> OAGI. This is a development-only convenience; shipping a shared password in source is not an acceptable production posture. Replace with a build-time injected value, a per-tenant secret, or remove the fallback entirely (back to the empty default that requires manual configuration). The `KVCredentialsProvider` already supports a missing `fallbackRecord` -- the deletion site is just `buildDefaultApi()` in `api.ts` plus the constant + tests in `credentials.ts` / `neon-credentials.test.ts`.
- [ ] **Code signing -- TeamIdentifier=not set + wrong designated requirement.** SIGNING-SPIKE confirmed the bug manifests; ADR-029 records the deferral. Lite ships with the same broken-but-functional signature full ships with today (right-click → Open one time, then auto-update works). Required before public launch:
  1. Resolve why codesign generates `certificate root = H"<leaf-hash>"` instead of the standard `anchor apple generic + leaf[OU] = "<TeamID>"` DR. Likely fix path: explicit `--keychain` and/or explicit `--requirements`. See `lite/SIGNING-SPIKE.md` "Findings" section.
  2. Once `codesign --verify --deep --strict` passes, flip `notarize: true` in `lite/electron-builder.json`
  3. Run a real `xcrun notarytool submit` round-trip and confirm the Apple notary accepts
  4. Verify with `spctl --assess --type execute --verbose` that Gatekeeper passes without prompts
  5. Set `RESIGN=1` as the default in `scripts/notarize.js` (currently opt-in)
  6. Back-port the fix to full (same `scripts/notarize.js`, just enable in full's `package.json` build config)
- [ ] **Windows code signing** -- deferred to Phase 1 (EV cert procurement is a prerequisite). Same as full.

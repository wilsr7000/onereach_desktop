üìñWhat is AI Run Times?

AI Run Times is an advanced RSS reader built into Onereach.ai that provides intelligent reading time estimation and progress tracking for UX Magazine articles. Unlike generic RSS readers, AI Run Times analyzes the actual content of each article to provide accurate reading time estimates.

‚ú®Key Features

üìäIntelligent Reading Time Estimation

Calculates accurate reading times (7-35 minutes) based on actual article content analysis, not generic estimates. The system analyzes word count and uses a 200 words per minute reading speed to provide realistic time estimates.

‚ö°Real-time Content Fetching

Uses Electron's native networking to bypass browser limitations and fetch full article content. This ensures reliable access to content without CORS restrictions.

üìàProgress Tracking

Visual progress bars show reading completion status with accurate time estimates. Track your reading progress with real-time updates.

üé®Modern Interface

Responsive grid layout with article previews, images, and clean typography. The interface updates dynamically with visual feedback when reading times are calculated.

üîßHow It Works

  1. RSS Feed Processing: Fetches the latest articles from UX Magazine's RSS feed
  2. Content Analysis: Downloads full article content and analyzes word count
  3. Smart Estimation: Calculates reading time using 200 words per minute average
  4. Dynamic Updates: Reading times update from "Loading..." to accurate estimates
  5. Visual Feedback: Green highlights indicate when reading times are updated
  6. Progress Tracking: Blue progress bars show reading completion (e.g., "2:30 / 14:00")

üî¨Technical Details

Reading Speed Calculation: AI Run Times uses a 200 words per minute reading speed, which is the average for technical content. The system strips HTML tags and counts actual words for accurate estimation.

Content Processing: Articles are processed using Electron's native net.request() module, which bypasses browser CORS limitations and provides reliable content fetching.

Real-time Updates: The system uses Inter-Process Communication (IPC) to send reading time updates from the main process to the renderer, ensuring the UI stays responsive while processing happens in the background.
